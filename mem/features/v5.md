- Generated images + DNA reference images live in IndexedDB (`src/lib/images.ts`), keyed: `scene:{topicId}:{n}`, `thumb:{topicId}:{i}`, `dna:{slot}`. localStorage only holds text/metadata.
- **Gemini-only mode (current)**: `normalizeSettings` in `src/lib/provider.ts` coerces any OpenAI routing (text/voice/image/thumbnail) to Gemini; defaults are text/voice=gemini, image/thumbnail=`gemini` (`gemini-2.5-flash-image`). OpenAI stays selectable in API Settings dropdown but is NEVER an active provider ("OpenAI Status: Disabled" in the diagnostics panel). Recraft/Puter remain selectable if explicitly chosen. Client helper `src/lib/generate-image.ts` builds prompts silently via `src/lib/style-lock.ts`, injecting Visual Instructions + AI Instructions + Visual DNA refs + the project's selected Visual Style.
- **Slow Safe Image Queue** (`src/routes/queue.tsx`): sequential, one image at a time, configurable delay (10/15/30/60s, default 15) via `src/lib/free-mode.ts` `getSafeDelaySec`. Buttons: Start Safe Queue, Pause/Resume, Continue From Last Scene, Generate One Image, Retry Current, Retry Failed. Counts: Completed, Pending, Rate Limited, Failed. On Gemini rate limit: NO long waiting — the scene is marked `rate-limited` (new QueueStatus, resumable, never `failed`), completed work is kept, and the queue pauses immediately with "Gemini image limit reached. Resume later." After ≥3 rate hits in one run it shows "Daily provider limit likely reached. Try again later or switch provider." Per-request 90s timeout (`IMAGE_TIMEOUT_MS`) guarantees no hang; `generate-image.ts` no longer does the multi-minute Free Mode retry loop — it surfaces rate limits immediately. Thumbnails (`thumbnail.tsx`): Free Mode caps `renderImages` to 1 image, hides Generate Alternatives, and stops on rate limit with the same message.
- **Puter AI** image provider (`src/lib/puter-image.ts`): browser-only SDK loaded via `<script src="https://js.puter.com/v2/">` in `__root.tsx` head. No API key. Selected as image choice `"puter"` in `provider.ts` (synthetic always-connected provider). `generate-image.ts` routes `puter` through the browser SDK (`puter.ai.txt2img`) instead of the server route, and auto-falls back to Gemini/Recraft (`fallbackImageProviderPayload`) when Puter is offline/rate-limited. Status (Connected/Generating/Rate Limited/Offline) tracked in local storage via `usePuterStatus`. Errors are logged, never crash the UI.
- Visual DNA (`src/lib/visual-dna.ts`) = 8 global reference slots incl. stickman character; applied to every project automatically.
- AI Instructions (`src/lib/instructions.ts`) = persistent art/story direction flattened into every generation prompt.
- Floating AI chat: `src/components/AIChat.tsx` + `POST /api/chat` (non-streaming, friendly producer persona, receives project context).
- Auto-Generate button on Projects runs research to story to storyboard+images to thumbnails+images to SEO to rating.
- Quality self-review lives in `reviewStage` in `ai.functions.ts` (auto-regenerates research/story below score 6).
